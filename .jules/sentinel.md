## 2025-05-14 - [CRITICAL] Secret Leakage and Hardcoded Credentials
**Vulnerability:** Found two security risks: (1) Vite was configured to inject `GEMINI_API_KEY` from the environment into the client-side bundle via the `define` key in `vite.config.ts`, making it visible to anyone inspecting the frontend. (2) Database connection details (host, user, dbName) were hardcoded as default state in `src/App.tsx`.
**Learning:** The API key injection was likely a leftover from earlier development of an AI feature that was later mostly removed (as noted in `DEADCODE.md`). The hardcoded database credentials were used as "convenience" defaults during development but leaked internal infrastructure details (like `localhost`, `root`, and specific DB names) and discouraged proper configuration.
**Prevention:** (1) NEVER use Vite's `define` to inject sensitive secrets into the frontend. Use a backend proxy for API calls that require secrets. (2) Use environment variables for sensitive configuration and provide a `.env.example` file instead of hardcoding defaults in source code.
